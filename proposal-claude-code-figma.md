# پروپوزال فنی: اتصال Claude Code به فیگما از طریق پلاگین پل‌ارتباطی (مسیر دوم)

**موضوع:** طراحی و ویرایش مستقیم فایل‌های فیگما توسط هوش مصنوعی، روی پلن رایگان، بدون Dev Mode و بدون API key
**رویکرد:** پل WebSocket میان Claude Code و یک پلاگین فیگما که از طریق Plugin API روی canvas می‌نویسد

---

## ۱. خلاصه‌ی مدیریتی

هدف این پروپوزال طراحی یک سامانه است که سه نیازمندی اصلی را برآورده کند:

۱. **Claude Code به‌سادگی به سامانه وصل شود** و به‌عنوان «مغز» تصمیم‌گیرنده عمل کند، بدون نیاز به کلید API جداگانه (احراز هویت از طریق اکانت Anthropic خودِ Claude Code انجام می‌شود).

۲. **پلاگین فیگما بتواند هر چیزی را بفهمد** (کل ساختار، محتوا، استایل‌ها و متغیرهای فایل را بخواند) **و هر چیزی را طراحی کند** (فریم، متن، کامپوننت، رنگ، auto-layout، توکن‌ها و... را بسازد و تغییر دهد).

۳. **اتصال پلاگین به Claude Code آسان باشد** — ترجیحاً یک‌کلیکی یا با حداقل کانفیگ دستی.

دلیل انتخاب این مسیر به‌جای MCP رسمی فیگما این است که در MCP رسمی، نوشتن روی canvas پشت seat پولی قفل است و پلن رایگان تنها حدود ۶ فراخوان در ماه دارد. در مسیر پل‌ارتباطی، نوشتنِ واقعی داخل خود فیگما و از طریق Plugin API انجام می‌شود؛ بنابراین کاملاً رایگان است و آن محدودیت‌ها را دور می‌زند.

---

## ۲. معماری کلی سامانه

جریان داده در این معماری به این شکل است:

```
Claude Code (CLI)  ⇄  سرور MCP محلی  ⇄  سرور WebSocket محلی  ⇄  پلاگین فیگما  ⇄  Figma Plugin API  ⇄  Canvas
       (مغز)            (مترجم ابزارها)        (پل پیام‌ها)         (مجری)          (موتور نوشتن)       (فایل)
```

نقش هر جزء:

- **Claude Code:** دستور زبان طبیعی کاربر را می‌گیرد، تصمیم می‌گیرد چه ابزاری را با چه پارامترهایی صدا بزند، و خروجی را تفسیر می‌کند. هوش و احراز هویت اینجاست.
- **سرور MCP محلی:** مجموعه‌ای از «ابزار»ها (tools) را به Claude Code معرفی می‌کند؛ مثل `get_document_info`، `create_frame`، `set_fill`، `set_text`. هر فراخوان ابزار را به یک پیام استاندارد تبدیل می‌کند.
- **سرور WebSocket محلی:** صرفاً یک رله است. پیام را از سرور MCP می‌گیرد و روی یک «کانال» مشخص به پلاگین می‌فرستد و نتیجه را برمی‌گرداند. این لایه است که اتصال دوطرفه‌ی بی‌درنگ را ممکن می‌کند.
- **پلاگین فیگما:** داخل ادیتور فیگما اجرا می‌شود، به کانال WebSocket وصل می‌شود، پیام‌ها را می‌گیرد و آن‌ها را به فراخوان‌های Plugin API ترجمه می‌کند.
- **Figma Plugin API:** تنها رابطی است که اجازه‌ی خواندن و نوشتن واقعی روی فایل را می‌دهد و روی همه‌ی پلن‌ها رایگان است.

نکته‌ی کلیدی: کل این زنجیره روی `127.0.0.1` (لوکال) کار می‌کند. هیچ داده‌ای به سرور بیرونیِ شخص ثالث نمی‌رود، و هیچ کلید API فیگمایی لازم نیست چون پلاگین مستقیماً داخل سشن احرازشده‌ی فیگما اجرا می‌شود.

## ۳. نیازمندی ۱ — اتصال آسان Claude Code

### ۳.۱. پیش‌نیازها

- **Node.js** نصب‌شده (نسخه‌ی LTS فعلی).
- **Claude Code** نصب‌شده با دستور `npm install -g @anthropic-ai/claude-code` و لاگین‌شده با اکانت Anthropic. (همین لاگین جای API key را می‌گیرد.)
- **اپ دسکتاپ فیگما** — نسخه‌ی مرورگر برای این کار کافی نیست، چون پلاگین باید داخل ادیتور دسکتاپ اجرا و به WebSocket محلی وصل شود.

### ۳.۲. مراحل اتصال

۱. سرور MCP پل را به‌صورت یک سرور HTTP/stdio محلی اجرا کن (یا از بسته‌ی آماده استفاده کن).
۲. آن را در Claude Code ثبت کن، نمونه:
   `claude mcp add --transport http talk-to-figma <URL سرور محلی>`
۳. Claude Code را کامل ببند و دوباره باز کن تا سشن جدید اتصال را تشخیص دهد.
۴. با دستور `/mcp` چک کن که سرور `talk-to-figma` در لیست ظاهر شده باشد.

### ۳.۳. معیار پذیرش

- اجرای `/mcp` نام سرور پل را با وضعیت «connected» نشان دهد.
- یک ابزار آزمایشی مثل «اطلاعات سند فعلی را بده» بدون خطا اجرا شود و خروجی واقعی از فایلِ بازِ فیگما برگرداند.
- هیچ مرحله‌ای نیاز به وارد کردن کلید API نداشته باشد.

## ۴. نیازمندی ۲ — پلاگین «همه‌چیز را بفهمد و همه‌چیز را طراحی کند»

این نیازمندی دو نیمه دارد: **فهمیدن (خواندن)** و **طراحی کردن (نوشتن)**. هر دو باید از طریق مجموعه‌ی کاملی از ابزارها روی Plugin API پوشش داده شوند.

### ۴.۱. نیمه‌ی فهم — خواندن کامل فایل

پلاگین باید این قابلیت‌ها را به‌صورت ابزار در اختیار Claude Code بگذارد:

- **پیمایش کل درخت سند:** خواندن صفحات، فریم‌ها، گروه‌ها و همه‌ی نودهای فرزند به‌صورت بازگشتی.
- **خواندن خصوصیات هر نود:** نوع، نام، موقعیت و اندازه، چرخش، شفافیت، قفل/مخفی بودن.
- **خواندن استایل‌ها:** fill، stroke، effect (سایه/بلور)، شعاع گوشه، blend mode.
- **خواندن تایپوگرافی:** محتوای متن، فونت، وزن، اندازه، فاصله‌ی خطوط و حروف.
- **خواندن چیدمان:** تنظیمات auto-layout، فاصله‌ها، padding، جهت، رفتار hug/fill.
- **خواندن سیستم طراحی:** کامپوننت‌ها و instanceها، متغیرها (variables) و توکن‌ها، استایل‌های اشتراکی.
- **خواندن انتخاب فعلی (selection):** تا کاربر بتواند بگوید «این را تغییر بده» و پلاگین بداند «این» چیست.
- **درک بصری:** خروجی‌گرفتن (export) از یک نود به‌صورت تصویر و فرستادن آن به Claude Code، تا مدل ظاهر را هم ببیند نه فقط داده‌ی ساختاری را.

نکته‌ی مهم درباره‌ی «هر چیزی را بفهمد»: مرز واقعی این است که پلاگین فقط می‌تواند **فایلِ بازِ فعلی** را بفهمد. به فایل‌های دیگرِ اکانت، به تنظیمات سطح‌حساب، یا به اطلاعات خارج از سند دسترسی ندارد. پس «فهم کامل» یعنی فهم کاملِ فایلِ جاری، نه کل فضای کاری.

### ۴.۲. نیمه‌ی طراحی — نوشتن کامل روی canvas

ابزارهای نوشتن باید این موارد را پوشش دهند:

- **ساخت اشیا:** فریم، مستطیل، بیضی، خط، چندضلعی، ستاره، vector.
- **متن:** ساخت لایه‌ی متن، تنظیم محتوا، فونت، اندازه، وزن، رنگ، تراز.
- **ظاهر:** تنظیم fill (تخت/گرادیان/تصویر)، stroke، شعاع گوشه، effectها.
- **چیدمان:** فعال‌کردن و تنظیم auto-layout، فاصله، padding، جهت، constraintها، گروه‌بندی و فریم‌بندی.
- **سیستم طراحی:** ساخت/درج instance از کامپوننت‌ها، ساخت و مقداردهی متغیرها و توکن‌ها، اعمال استایل‌های اشتراکی.
- **تصویر:** ایجاد fill تصویری از داده‌ی تصویر (`createImage`).
- **عملیات ساختاری:** جابه‌جایی، تغییر اندازه، کپی، حذف، ترتیب لایه‌ها، عملیات بولین (union/subtract).

### ۴.۳. الزامات فنی حیاتی (بدون این‌ها پلاگین می‌شکند)

این بخش قلب «عمیق» بودنِ پروپوزال است؛ این نکات همان جاهایی است که پیاده‌سازی‌های ساده در آن گیر می‌کنند:

- **بارگذاری فونت پیش از نوشتن متن:** قبل از تنظیم یا تغییر هر متن، باید `figma.loadFontAsync` برای فونتِ دقیق آن لایه فراخوانی شود؛ وگرنه عملیات خطا می‌دهد. (مثلاً برای فونت Inter، نام وزن «Semi Bold» است نه «SemiBold».)
- **دسترسی شبکه در manifest:** در `manifest.json` باید دامنه‌ی سرور WebSocket محلی زیر `networkAccess.allowedDomains` مجاز شود، وگرنه پلاگین اجازه‌ی اتصال نمی‌گیرد.
- **سریال‌سازی نودها:** نودهای فیگما را نمی‌توان مستقیم در پیام JSON فرستاد. باید با **شناسه‌ی نود (id)** ارجاع داد و در سمت پلاگین با `figma.getNodeById` بازیابی کرد.
- **اجرای غیرمسدودکننده و دسته‌ای (batching):** عملیات‌های سنگین باید به‌صورت ناهمگام و دسته‌ای انجام شوند تا رابط فیگما فریز نشود؛ روی فایل‌های بزرگ این تفاوت میان «کار می‌کند» و «هنگ می‌کند» است.
- **گزارش خطای ساختاریافته:** هر فرمان باید نتیجه‌ی موفق یا پیام خطای روشن را به Claude Code برگرداند تا مدل بتواند خودش را اصلاح کند (مثلاً «فونت پیدا نشد» → مدل فونت دیگری انتخاب کند).
- **بازیابی اتصال:** اگر WebSocket قطع شد، پلاگین باید خودکار دوباره وصل شود و وضعیت کانال را حفظ کند.

### ۴.۴. معیار پذیرش

- Claude Code بتواند یک فریم خالی را از صفر بسازد، داخلش متن با فونت دلخواه بگذارد، رنگ بدهد و auto-layout اعمال کند — همه با یک دستور زبان طبیعی.
- Claude Code بتواند یک نودِ موجود را «بفهمد» (خصوصیاتش را گزارش دهد) و بعد دقیقاً همان را اصلاح کند.
- ویرایش متن بدون خطای فونت انجام شود.
- روی یک فایل با چند صد لایه، عملیات بدون فریزشدنِ فیگما تمام شود.

## ۵. نیازمندی ۳ — اتصال آسان پلاگین به Claude Code

### ۵.۱. مکانیزم جفت‌سازی (channel)

برای اینکه پلاگین و سرور MCP بدانند با هم حرف می‌زنند، یک مفهوم «کانال» لازم است: پلاگین هنگام باز شدن یک شناسه‌ی کانال نشان می‌دهد، و Claude Code با همان شناسه به کانال «join» می‌کند. این کار از قاطی‌شدن چند سشن جلوگیری می‌کند و اتصال را قابل‌اعتماد می‌سازد.

### ۵.۲. تجربه‌ی راه‌اندازی هدف (آسان‌سازی)

برای رسیدن به «اتصال آسان»، تجربه‌ی ایده‌آل این است:

- یک **اپ/سرور محلی با دکمه‌ی «Start Server»** که با یک کلیک رله‌ی WebSocket را بالا می‌آورد و در پس‌زمینه روشن می‌ماند.
- **کانفیگ آماده‌ی کپی‌-چسباندن** برای ثبت سرور در Claude Code، بدون نیاز به نوشتن دستی JSON پیچیده.
- پلاگین هنگام باز شدن، **به‌صورت خودکار** به کانال محلی وصل شود و وضعیت «connected» را نشان دهد.

### ۵.۳. معیار پذیرش

- کل مسیر اتصال (روشن‌کردن سرور، باز کردن پلاگین، جوین‌کردن Claude Code) زیر چند دقیقه و بدون ویرایش دستیِ فایل کانفیگ انجام شود.
- پس از یک‌بار راه‌اندازی، دفعات بعد فقط با باز کردن پلاگین و روشن‌کردن سرور کار کند.

## ۶. نقشه‌ی راه مرحله‌به‌مرحله

مرحله ۱ — **آماده‌سازی محیط:** نصب Node.js، نصب و لاگین Claude Code، نصب اپ دسکتاپ فیگما.

مرحله ۲ — **بالا آوردن لایه‌ی پل:** اجرای سرور MCP و سرور WebSocket محلی (یا اجرای بسته‌ی آماده با دکمه‌ی Start Server).

مرحله ۳ — **نصب پلاگین فیگما:** افزودن پلاگین به‌صورت development plugin از طریق منوی Plugins → Development → Import plugin from manifest، با manifestِ دارای `networkAccess` صحیح.

مرحله ۴ — **ثبت سرور در Claude Code:** اجرای دستور `claude mcp add`، سپس ری‌استارت و بررسی با `/mcp`.

مرحله ۵ — **جفت‌سازی کانال:** باز کردن پلاگین داخل فایل هدف، گرفتن شناسه‌ی کانال، و جوین‌کردن Claude Code به همان کانال.

مرحله ۶ — **آزمون دود (smoke test):** اجرای یک دستور خواندن (فهم) و یک دستور نوشتن ساده (طراحی) برای تأیید کارکرد دوطرفه.

مرحله ۷ — **استفاده‌ی واقعی:** دادن دستورهای طراحی پیچیده‌تر و پایش گزارش خطاها.

## ۷. امنیت و حریم خصوصی

- کل ارتباط روی `127.0.0.1` بماند؛ سرور WebSocket نباید روی شبکه‌ی عمومی باز شود.
- هیچ کلید API یا توکن حساسی در کد پلاگینِ منتشرشده قرار نگیرد. برای استفاده‌ی شخصی، اجرای آن به‌صورت development plugin امن‌تر است.
- پلاگین با دسترسی کامل به فایل اجرا می‌شود؛ پیش از عملیات‌های مخرب (حذف، بازنویسی انبوه) بهتر است یک گام تأیید یا یک حالت «خواندن‌-تنها» وجود داشته باشد.
- چون نوشتن از طریق Plugin API انجام می‌شود و نه سرویس ابری، داده‌ی طراحی از کامپیوتر خارج نمی‌شود (به‌جز آنچه عمداً به‌عنوان context به مدل فرستاده می‌شود).

## ۸. ریسک‌ها و محدودیت‌ها

- **ماهیت غیررسمی:** این مسیر بر پایه‌ی ابزارهای جامعه‌ای است و رسمیِ فیگما نیست؛ به‌روزرسانی‌های فیگما ممکن است گاهی آن را بشکنند.
- **مرز Plugin API:** بعضی کارها (برخی قابلیت‌های Dev Mode، عملیات سطح‌حساب، دسترسی به فایل‌های بازنشده) از طریق Plugin API ممکن نیست؛ پس «هر چیزی» در عمل یعنی «هر چیزی که Plugin API اجازه می‌دهد».
- **کارایی روی فایل‌های بسیار بزرگ:** بدون batching درست، فیگما کند یا فریز می‌شود.
- **وابستگی به فایلِ باز:** پلاگین فقط فایلی را می‌بیند که در ادیتور باز است.
- **کیفیت خروجی:** «فهمِ» مدل به کیفیت داده‌ای که پلاگین می‌فرستد بستگی دارد؛ فرستادن هم‌زمانِ ساختار + تصویرِ export شده کیفیت تصمیم‌گیری را بالا می‌برد.
